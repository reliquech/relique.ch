  <action>
Change `fetchList` to accept optional `{ force?: boolean }`.
Behavior:
- If cache entry exists and `force !== true`, set cached data immediately.
- If cache entry exists and is fresh, still allow background refresh but do not set `loading=true`.
- If no cache entry exists, set `loading=true`.
- Always set `refreshing=true` during background fetch when cached data exists.
- On successful response, call `writeMarketplaceItemsCache(queryKey, response)` and update state only if the active key still matches.
- On error with cached data, keep cached data and set `error` to the error message.
- On abort, do not set error or loading/refreshing incorrectly.
  </action>
