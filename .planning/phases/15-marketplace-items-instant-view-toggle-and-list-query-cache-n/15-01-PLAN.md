<objective>
Extract deterministic Marketplace Items list query-key and cache primitives so presentation-only view changes cannot affect data fetching.
</objective>

<threat_model>
T15-01: Cache keys that include presentation state cause accidental refetches. The key helper must include only data-affecting params and explicitly exclude `density` and view.
T15-02: Unbounded module cache can leak memory in long admin sessions. Add a small max-entry cap.
</threat_model>
