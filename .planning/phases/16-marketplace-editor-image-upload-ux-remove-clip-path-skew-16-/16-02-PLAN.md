# Plan 16-02: Native Drag and Drop Reordering for Additional Images

## Context
Phase 16 focuses on improving the image upload UX in the marketplace editor. This plan covers:
1. Adding native HTML5 Drag and Drop capabilities to the additional images grid in the marketplace editor.
2. Synchronizing the state changes from drag and drop actions back to React Hook Form's `images` value so the visual order matches the database payload.

## Tasks

### Task 16-02-01: Implement Reordering Utility in hook
- **Files**:
  - `src/features/marketplace/hooks/useMarketplaceFormImages.ts`
- **Action**:
  - Expose a `reorderAdditionalImages(activeIndex: number, overIndex: number)` function.
  - Inside the helper, slice and splice the `additionalImages` array to swap or insert the item at the `overIndex`.
  - Update `additionalImages` state and `additionalImagesRef.current`.
  - Call `syncAdditionalUrls` with the newly ordered list to update the React Hook Form field `images`.
- **Verify**:
  - Validate hook code changes compile successfully.
  - Command: `npm run typecheck`
- **Done**:
  - Reordering function is implemented and exposed by the hook.

### Task 16-02-02: Integrate Drag and Drop Interactions in UI Component
- **Files**:
  - `src/components/admin/marketplace/MarketplaceImageSection.tsx`
- **Action**:
  - Make additional image tiles draggable using HTML5 `draggable={true}`.
  - Implement handlers: `onDragStart` (stores source index using `e.dataTransfer.setData("text/plain", index)`), `onDragOver` (calls `e.preventDefault()`), and `onDrop` (retrieves the source index, target index, and calls `reorderAdditionalImages(srcIndex, destIndex)`).
  - Apply visual indicators: change cursor to `cursor-grab` (or `cursor-grabbing` while dragging) and reduce opacity / add active borders during drag events.
- **Verify**:
  - Verify compile status and linting.
  - Command: `npm run typecheck && npm run lint`
- **Done**:
  - Native drag-and-drop mechanics implemented, visually styled, and integrated.
