# Plan 16-01: Skew Removal, Layout Adjustments, and Image Lightbox

## Context
Phase 16 focuses on improving the image upload UX in the marketplace editor. This plan covers:
1. Removing the brand slant clip-path and skew properties from the upload components.
2. Converting the cover preview to a clean 16:9 ratio and the additional images to a square grid (2/3/4 columns responsive).
3. Implementing the retry logic for failed uploads in the `useMarketplaceFormImages` hook.
4. Designing and wiring a reusable `ImageLightbox` component with keyboard navigation, body scroll lock, index tracker, and focus trap.

## Tasks

### Task 16-01-01: Expose Retry Hook Logic and Clean Up Object URLs
- **Files**:
  - `src/features/marketplace/hooks/useMarketplaceFormImages.ts`
- **Action**:
  - Add a `retryUpload(id: string)` method that finds the failed item by ID in `coverImage` or `additionalImages`.
  - If found in `coverImage`, retrieve the cached `file` reference, transition state to `uploading`, and call `uploadMarketplaceFile`.
  - If found in `additionalImages`, transition that item's state to `uploading` and call `uploadMarketplaceFile` for it.
  - Ensure all Object URLs are properly tracked and cleaned up on file swap, removal, or component unmount.
- **Verify**:
  - Validate that the file compiles without errors.
  - Command: `npm run typecheck`
- **Done**:
  - `retryUpload` helper is implemented and exposed.

### Task 16-01-02: Create Reusable Image Lightbox Component
- **Files**:
  - `src/components/shared/ImageLightbox.tsx` (New file)
- **Action**:
  - Implement a Modal-based Lightbox using standard HTML/React elements.
  - Attributes: backdrop overlay with dark tint and blur (`bg-black/90 backdrop-blur-md`).
  - Listeners: Keyboard Escape to close, ArrowLeft / ArrowRight to paginate.
  - Accessibility: trapped focus (loop tabs between buttons), body scroll lock (add/remove `overflow-hidden` class on document body), explicit `aria-label` tags for controls, display current index (e.g. `2 / 5`).
- **Verify**:
  - Validate that the new lightbox component compiles and has correct types.
  - Command: `npm run typecheck`
- **Done**:
  - Reusable `ImageLightbox.tsx` file created and verified.

### Task 16-01-03: Remove Clip-Path, Refactor Grid & Wire Lightbox and Retry UI
- **Files**:
  - `src/components/admin/marketplace/MarketplaceImageSection.tsx`
- **Action**:
  - Remove all `clip-path-slant` and `clip-path-slant-lg` CSS classes.
  - Apply `aspect-video` (16:9) to the Cover Image container.
  - Apply `grid grid-cols-2 sm:grid-cols-3 md:grid-cols-4 gap-4` to the Additional Images container.
  - Style Additional Image items with `aspect-square`.
  - Render a small hover action or top-right delete button (`w-8 h-8 rounded-full`) instead of skewed buttons.
  - Under failed items, display the error state with a clickable `Retry` button calling `retryUpload(item.id)`.
  - Bind click events on successfully uploaded previews to trigger the Lightbox, displaying the active image index.
- **Verify**:
  - Run build and lint checks to confirm clean build.
  - Command: `npm run typecheck && npm run lint`
- **Done**:
  - Component updated, all visual skew removed, aspect ratios aligned, and lightbox wired.
