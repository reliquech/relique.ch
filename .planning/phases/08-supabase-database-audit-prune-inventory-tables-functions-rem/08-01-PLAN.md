Create `scripts/audit-supabase-usage.mjs` (Node ESM, no new deps) that:

1. Recursively scans `src/**/*.{ts,tsx}` for:
   - `.from("table_name")` and `.from('table_name')`
   - `.rpc("fn_name")` and `.rpc('fn_name')`
   - Storage bucket refs: `.from("bucket")` on `supabase.storage` (pattern `storage.from(`)

   - `create table public.{name}`
   - `create or replace function public.{name}`
   - `insert into storage.buckets` / bucket id strings
3. Outputs structured JSON to stdout AND writes `supabase/SUPABASE_USAGE.md` with sections:
   - **Tables** — columns: Object | Migrations | App refs (file:line list) | Verdict (KEEP/PRUNE)
   - **RPC/Functions** — same format
   - **Storage buckets** — same format
   - **Triggers-only functions** (`handle_updated_at`, `handle_new_user`, `log_audit_event`, `set_marketplace_state_timestamps`) — verdict MERGE_IN_BASELINE

Apply locked verdicts regardless of grep hits:
- `email_logs` → **PRUNE** (note: "Resend removed — locked D-08-01")
- `admin_upsert_profile` → **PRUNE** (note: "unused RPC — locked D-08-02")
- 7 dashboard RPCs → **KEEP**
- 3 buckets → **KEEP**

For tables with zero app refs but active admin API (e.g. `pipeline_stages`), mark **KEEP** if any `/api/` route references them.

Do NOT mark `crm_recent_searches` PRUNE — `src/app/api/crm-searches/route.ts` uses it.

Script must be idempotent and runnable via `node scripts/audit-supabase-usage.mjs`.
