<objective>
Create editor state infrastructure for autosave, stale-response protection, draft create/update service calls, and unsaved-change guards.
</objective>

<threat_model>
T13-06: Slow autosave responses can overwrite newer edits. Autosave must use a monotonic request sequence and ignore stale responses.
T13-07: Dirty editor navigation can lose data. The guard must register `beforeunload` only when dirty and expose a confirm path for internal navigation.
T13-08: New draft autosave can create duplicate drafts if called repeatedly before id assignment. The service/hook must serialize create-draft behavior and reuse the created id.
</threat_model>

Create `useMarketplaceAutosave.ts`.
Export `type AutosaveState = "idle" | "dirty" | "saving" | "saved" | "failed" | "offline"`.
Hook props:
`itemId?: string`, `values: MarketplaceFormData`, `dirty: boolean`, `online: boolean`, `enabled: boolean`, `uploading: boolean`, `onDraftCreated?: (id: string) => void`.
Debounce save by 1000ms.
Use `requestSeqRef` and increment before each save. Only apply success/failure state if the response sequence equals the latest sequence.
If `!online`, state is `offline`.
If `uploading`, do not autosave.
If no `itemId`, call `createMarketplaceDraft`, then call `onDraftCreated(result.id)`.
If `itemId`, call `updateMarketplaceItem`.
Return `{ state, retry, flush, lastSavedAt, error }`.
